# Комплексный план проверки конвертации данных SEAF1 и SEAF2

Цель: Подтвердить корректность и целостность данных при двухсторонней конвертации (YAML <-> Excel) для обоих стандартов, а также верифицировать результат через сравнение схем DrawIO.

---

## 1. Проверка SEAF1 (Единственное число)
**Источник данных:** Папка `seaf-dzo-example/architecture/ta/home_cinema`

### 1.1. Построение эталонной схемы
- **Действие:** Запуск `seaf2drawio.py` на базе исходных YAML файлов из `home_cinema`.
- **Результат:** Файл `original_seaf1.drawio`.

### 1.2. Этап: Экспорт SEAF1 (YAML -> XLSX)
- **Инструмент:** `seaf2xlsx/yaml_to_xlsx.py`.
- **Результат:** Генерация Excel-файлов в папку `xlsx_out`.

### 1.3. Этап: Импорт SEAF1 (XLSX -> YAML)
- **Инструмент:** `seaf2xlsx/xlsx_to_yaml.py`.
- **Результат:** Восстановленные YAML-файлы в папке `roundtrip_test/yaml_recovered`.

### 1.4. Построение схемы из восстановленных данных
- **Действие:** Запуск `seaf2drawio.py` на базе файлов из `yaml_recovered`.
- **Результат:** Файл `recovered_seaf1.drawio`.

### 1.5. Верификация и сравнение
- **Сравнение размеров:** Размер `original_seaf1.drawio` и `recovered_seaf1.drawio` не должен существенно различаться.
- **Контроль:** Проверка логов `Verification summary`.

## 2. Проверка SEAF2 (Множественное число)
**Источник данных:** Папка `ta`

### 2.1. Этап: Экспорт SEAF2 (YAML -> XLSX)
- **Инструмент:** `seaf2xlsx/_seaf2_yaml_to_xlsx_ta.py`.
- **Ожидаемый результат:** Создание Excel-файлов в папке `ta_xlsx`.

### 2.2. Этап: Импорт SEAF2 (XLSX -> YAML)
- **Инструмент:** `seaf2xlsx/_seaf2_xlsx_to_yaml.py`.
- **Ожидаемый результат:** Восстановленные YAML-файлы в папке `ta_recovered` (множественное число, префикс `company`).

### 2.3. Визуализация SEAF2 (Схема DrawIO)
- **Статус:** [НЕ РЕАЛИЗОВАНО / ОТЛОЖЕНО]
- **Причина:** Требуется поддержка множественного числа в генераторе схем или создание суррогатных файлов.
